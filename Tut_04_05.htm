Key Components Explained

1. XR Origin (Root GameObject)

Component: XR Origin

Purpose: Defines the "origin point" of your VR space. The camera and controllers are positioned relative to this origin. When you move the XR Origin, the entire player (camera + controllers) moves.

Key Settings:

  • Camera Floor Offset Object: Reference to Camera Offset (for height adjustments)
  • Requested Tracking Mode: Usually "Room Scale"

2. Camera Offset

Purpose: Container for the camera and controllers. This allows you to adjust player height or position all player components together.

Why it exists: In VR, the headset position (0,0,0) might be at floor level. Camera Offset lets you raise the player to standing height (typically Y = 1.36 meters).

3. Main Camera

Components:

  • Camera: Standard Unity camera - this is what renders the VR view
  • Tracked Pose Driver: Receives position/rotation from the VR headset
  • Audio Listener: Receives 3D audio (where you "hear" from)

Key Point: The Tracked Pose Driver automatically updates the camera position based on where the player moves their head in real space.

https://gyazo.com/534a8b8ed9aad6beadc3b1c620086e54

4. LeftHand / RightHand Controllers

üÜï XRIT 3.0 Architecture: Near-Far Interactors

What Changed: In XRIT 2.x, you needed TWO separate interactors per hand:

  • ‚úó XR Direct Interactor (for touch-based grabbing)
  • ‚úó XR Ray Interactor (for distance pointing)
  • ‚úó ActionBasedController (for input - now deprecated)

XRIT 3.0 Solution: Near-Far Interactor - ONE component that does both!

  • ‚úì Automatically grabs nearby objects (direct mode)
  • ‚úì Automatically points at distant objects (ray mode)
  • ‚úì Has built-in Input Readers (no manual setup!)
  • ‚úì Intelligent mode switching based on distance

2 Inspect a Hand Controller

  4. Examine the settings:

5. XR Interaction Manager

Purpose: The "traffic controller" for all XR interactions in your scene.

What it does:

  • Manages communication between Interactors (controllers) and Interactables (grabbable objects)
  • Handles hover, select, and activate events
  • Resolves conflicts when multiple interactors target the same object
  • Processes Input Reader data in XRIT 3.0

Important: Every scene needs exactly ONE XR Interaction Manager. The Complete Rig prefab includes it automatically.

üîß PART 3: BUILD YOUR OWN CUSTOM RIG 30-50 MIN

üéØ Goal: Create a custom XR rig from scratch using XRIT 3.0 components

Why build manually? Understanding the manual process gives you full control for custom projects and helps troubleshooting.

Method 1: Minimal XR Origin + Near-Far Interactors (RECOMMENDED)

1 Create a New Scene

  1. File ‚Üí New Scene
  2. Save as "CustomRig_Scene"
  3. Delete the Main Camera (we'll create our own XR camera)

2 Add Minimal XR Origin

  1. Right-click in Hierarchy
  2. Select XR ‚Üí XR Origin (VR)

‚ö†Ô∏è XRIT 3.0 Note: This creates a MINIMAL rig - just the XR Origin, Camera Offset, and Main Camera. NO controllers are included! We'll add them manually.

3 Add Near-Far Interactors for Hands

  1. In Hierarchy, expand XR Origin
  2. Right-click on Camera Offset
  3. Select XR ‚Üí Interaction ‚Üí Near-Far Interactor
  4. Rename to "LeftHand Near-Far"
  5. Repeat steps 2-4 to create "RightHand Near-Far"

4 Configure Each Hand

For LeftHand Near-Far:

  1. Select it in Hierarchy
  2. In Inspector, find Near-Far Interactor component
  3. Set Controller Node to "Left Hand"

For RightHand Near-Far:

  1. Select it in Hierarchy
  2. Set Controller Node to "Right Hand"

üéâ That's it! Your custom rig is functionally complete. The Near-Far Interactors have built-in Input Readers, so no manual Input Action setup is required!

5 Add Visual Feedback (Optional but Recommended)

  1. Right-click LeftHand Near-Far in Hierarchy
  2. Select 3D Object ‚Üí Sphere
  3. Rename to "HandVisual"
  4. Set Transform:
    • Position: (0, 0, 0.05) - slightly forward
    • Scale: (0.05, 0.05, 0.05) - small sphere
  5. Repeat for RightHand Near-Far

6 Add Test Environment

  1. Create a Plane (floor) at (0, 0, 0)
  2. Create a Cube at (0, 1, 2)
  3. Add XR Grab Interactable component to the cube

7 Test Your Custom Rig

  1. Press Play
  2. Use XR Device Simulator controls (RMB for camera, Shift/Space for hands)
  3. Press G when near the cube to grab it
  4. Move hand away from objects and press Q or E to see ray mode activate automatically

üí° What You Just Experienced: Intelligent Mode Switching

The Near-Far Interactor automatically:

  • Near Mode: When your hand is close to an object, it uses direct grab (sphere collider)
  • Far Mode: When no objects are nearby, it switches to ray pointer
  • Smooth Transition: The switch happens seamlessly based on distance

This is why XRIT 3.0's Near-Far Interactor is so powerful - you get both interaction modes in ONE component!

Method 2: Advanced - Using XR Direct + Ray Interactors (Legacy Architecture)

üéì Educational Context: The XRIT 2.x Way

Why show this? Many existing projects and tutorials use separate Direct and Ray Interactors. Understanding this helps you work with older codebases.

Note: This method works in XRIT 3.0 but is considered legacy. Use Near-Far Interactor for new projects.

Instead of Near-Far Interactors, add separate components:

  1. Right-click Camera Offset
  2. Select XR ‚Üí Interaction ‚Üí XR Direct Interactor (Action Based)
  3. Rename to "LeftHand Direct"
  4. Right-click Camera Offset again
  5. Select XR ‚Üí Interaction ‚Üí XR Ray Interactor (Action Based)
  6. Rename to "LeftHand Ray"
  7. Repeat for right hand
https://gyazo.com/99ee758f24d00f75687ca86bd3300c57

Challenges with this approach:

  • Two components per hand to manage (4 total)
  • Potential conflicts between Direct and Ray (which takes priority?)
  • More complex configuration
  • Still needs Input Reader configuration in XRIT 3.0

Solution: Use Near-Far Interactor instead - it handles all of this automatically!
