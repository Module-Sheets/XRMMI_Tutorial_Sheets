Tutorial 01-02 Overview: Build XR Control Scene from Scratch

Overview Only: Building an XR Control scene from scratch with manual setup and then switching to automated XRIT systems.

Part 1: Manual XR Control Scene Setup

1. Build XR Control scene from scratch

  1. Import OpenXR in Package Manager Window/unity registry dropdown
    1. Restarts using New Input System - accept
  2. Top Menu/Edit/Project Settings/XR Plug-In Management/OpenXR:
    1. XR Plug-In Management: Tick OpenXR
    2. XR Plug-In Management/OpenXR:
      1. Add Khronos Simple Controller Profile
      2. Enable Mock Runtime (only required for custom mock controller, not XRIT mock controller)
  3. Convert Main Camera to XRRig
  4. Add Hand Controller objects (spheres with empty objects as pivot/parent)
  5. Add Tracked Pose Driver Component to each sphere
    1. Select Generic XR Controller
    2. Select Left or Right controller
    3. Not capable of taking proper mock inputs
  6. Add Simulated Control system â€“ Script
    1. Add MockController.cs Script from BB to Camera Offset
    2. Create own Input Actions Game Object and Mouse binding
    3. Mock Controller Script and TPD map Input Actions to Controllers
    4. Next we do it with Unity XRIT automated systems

Part 2: Switch to New XRIT Simulated Control System (XR Device Simulator)

2. Switch to new XRIT Simulated control system

  1. Package Window: Add XRIT, including Starter Assets, and XR Device Simulator
    1. If new install, decline the new interaction layer mask upgrade? (Edit:ProjectSettings:XRIT:)
  2. Controllers: Add Tracked Pose Driver Script (not component as in Week 1)
    1. Add bindings for device rotation
    2. Add bindings for device position
  3. Project Settings: Disable mock runtime
  4. Add XR Device Simulator prefab located in:
    1. Assets\Samples\XR Interaction Toolkit\2.0.4\XR Device Simulator
  5. Press play to test
Note: This transition from manual mock controller setup to automated XRIT systems demonstrates both approaches. The manual method provides understanding of the underlying systems, while XRIT automation provides a faster, more standardized workflow for future projects.
