  • ActionBasedController DEPRECATED: Old component removed - replaced by Input Readers

    Key Components Explained

    1. XR Origin (Root GameObject)

    Component: XR Origin

    Purpose: Defines the "origin point" of your VR space. The camera and controllers are positioned relative to this origin. When you move the XR Origin, the entire player (camera + controllers) moves.

    Key Settings:

    • Camera Floor Offset Object: Reference to Camera Offset (for height adjustments)
    • Requested Tracking Mode: Usually "Room Scale"

    2. Camera Offset

    Purpose: Container for the camera and controllers. This allows you to adjust player height or position all player components together.

    Why it exists: In VR, the headset position (0,0,0) might be at floor level. Camera Offset lets you raise the player to standing height (typically Y = 1.36 meters).

    3. Main Camera

    Components:

    • Camera: Standard Unity camera - this is what renders the VR view
    • Tracked Pose Driver: Receives position/rotation from the VR headset
    • Audio Listener: Receives 3D audio (where you "hear" from)

    Key Point: The Tracked Pose Driver automatically updates the camera position based on where the player moves their head in real space.

    https://gyazo.com/534a8b8ed9aad6beadc3b1c620086e54

    4. LeftHand / RightHand Controllers

    üÜï XRIT 3.0 Architecture: Near-Far Interactors

    What Changed: In XRIT 2.x, you needed TWO separate interactors per hand:

    • ‚úó XR Direct Interactor (for touch-based grabbing)
    • ‚úó XR Ray Interactor (for distance pointing)
    • ‚úó ActionBasedController (for input - now deprecated)

    XRIT 3.0 Solution: Near-Far Interactor - ONE component that does both!

    • ‚úì Automatically grabs nearby objects (direct mode)
    • ‚úì Automatically points at distant objects (ray mode)
    • ‚úì Has built-in Input Readers (no manual setup!)
    • ‚úì Intelligent mode switching based on distance

    2 Inspect a Hand Controller

    3. Find the Near-Far Interactor component (or XR Direct/Ray if using legacy setup)
    4. Examine the settings:

    Near-Far Interactor Component Breakdown:

    Section What It Does
    Input Configuration Input Readers embedded here!
    ‚Ä¢ Select Action Reader (trigger button)
    ‚Ä¢ Activate Action Reader (grip button)
    ‚Ä¢ UI Press Action Reader (primary button)
    Note: No manual Input Action references needed - it's automatic!
    Controller Node Which hand this interactor controls:
    ‚Ä¢ Left Hand
    ‚Ä¢ Right Hand
    Near Interaction ‚Ä¢ Sphere Collider (for detecting nearby objects)
    ‚Ä¢ Attach Transform (where grabbed objects snap to)
    ‚Ä¢ Collision radius (how close you need to be)

    ‚Ä¢ Ray Length (how far you can point)

    ‚Ä¢ Ray color and width settings

    ‚Ä¢ Smooth transitions between near and far

    üí° Key Insight: Notice how you don't see any Input Action references to copy/paste? That's because Input Readers handle everything automatically in XRIT 3.0. This is the major architectural improvement!

    5. XR Interaction Manager

    Purpose: The "traffic controller" for all XR interactions in your scene.

    What it does:

    • Manages communication between Interactors (controllers) and Interactables (grabbable objects)
    • Handles hover, select, and activate events
    • Resolves conflicts when multiple interactors target the same object
    • Processes Input Reader data in XRIT 3.0

    Important: Every scene needs exactly ONE XR Interaction Manager. The Complete Rig prefab includes it automatically.

    What to Do When You Encounter Legacy Projects

    Scenario 1: You Find a Tutorial Using ActionBasedController

    What you'll see in the tutorial:

    • Instructions to copy XR Controller component values
    • Pasting Input Action references into ~10 fields
    • Manual binding of Position, Rotation, Select, Activate actions

    What you should do instead:

    • ‚úì Use the Complete Rig prefab from Starter Assets
    • ‚úì Or use Near-Far Interactors with automatic Input Readers
    • ‚úì Skip the entire copy/paste workflow

    Scenario 2: Opening an Existing XRIT 2.x Project

    What happens: Unity will prompt you to upgrade to XRIT 3.0

    What you should know:

    • Unity provides automatic migration tools
    • ActionBasedController ‚Üí Input Readers (automatic)
    • Namespaces update automatically
    • Most functionality preserved

    Recommendation:

    • Back up your project first!
    • Let Unity perform automatic migration
    • Test thoroughly after migration
    • Consider refactoring to Near-Far Interactors for cleaner code

    Solution: XRIT 3.0 supports both legacy and modern approaches!
