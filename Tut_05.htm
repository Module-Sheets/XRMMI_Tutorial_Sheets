Key Components Explained

1. XR Origin (Root GameObject)

Component: XR Origin

Purpose: Defines the "origin point" of your VR space. The camera and controllers are positioned relative to this origin. When you move the XR Origin, the entire player (camera + controllers) moves.

Key Settings:

  • Camera Floor Offset Object: Reference to Camera Offset (for height adjustments)
  • Requested Tracking Mode: Usually "Room Scale"

2. Camera Offset

Purpose: Container for the camera and controllers. This allows you to adjust player height or position all player components together.

Why it exists: In VR, the headset position (0,0,0) might be at floor level. Camera Offset lets you raise the player to standing height (typically Y = 1.36 meters).

3. Main Camera

Components:

  • Camera: Standard Unity camera - this is what renders the VR view
  • Tracked Pose Driver: Receives position/rotation from the VR headset
  • Audio Listener: Receives 3D audio (where you "hear" from)

Key Point: The Tracked Pose Driver automatically updates the camera position based on where the player moves their head in real space.

https://gyazo.com/534a8b8ed9aad6beadc3b1c620086e54

4. LeftHand / RightHand Controllers

üÜï XRIT 3.0 Architecture: Near-Far Interactors

What Changed: In XRIT 2.x, you needed TWO separate interactors per hand:

  • ‚úó XR Direct Interactor (for touch-based grabbing)
  • ‚úó XR Ray Interactor (for distance pointing)
  • ‚úó ActionBasedController (for input - now deprecated)

XRIT 3.0 Solution: Near-Far Interactor - ONE component that does both! (Not in Rig yet, must drag prefab in to use)

  • ‚úì Automatically grabs nearby objects (direct mode)
  • ‚úì Automatically points at distant objects (ray mode)
  • ‚úì Has built-in Input Readers (no manual setup!)
  • ‚úì Intelligent mode switching based on distance

5. XR Interaction Manager

Purpose: The "traffic controller" for all XR interactions in your scene.

What it does:

  • Manages communication between Interactors (controllers) and Interactables (grabbable objects)

  • Resolves conflicts when multiple interactors target the same object
  • Processes Input Reader data in XRIT 3.0

Important: Every scene needs exactly ONE XR Interaction Manager. The Complete Rig prefab includes it automatically.

üí° What You Just Experienced: Intelligent Mode Switching

The Near-Far Interactor automatically:

  • Near Mode: When your hand is close to an object, it uses direct grab (sphere collider)
  • Far Mode: When no objects are nearby, it switches to ray pointer
  • Smooth Transition: The switch happens seamlessly based on distance

This is why XRIT 3.0's Near-Far Interactor is so powerful - you get both interaction modes in ONE component!

XR Interaction Setup Prefab - Complete Architecture

The XR Interaction Setup prefab contains five main systems:

  1. Input Action Manager - Centralized input configuration
  2. XR Input Modality Manager - Interaction event coordination
  3. Character Controller - Movement and Collision handling
  4. XR Origin (XR Rig) - Complete player rig with controllers
  5. Gaze System - Gaze-based interaction support
  6. Locomotion System - Movement, turning, climbing, teleportation

System 1: Input Action Manager

Component: Input Action Manager

Purpose: Manages the XRI Input Actions asset, enabling/disabling action maps globally. This is the central hub for all VR input in the scene.

System 2: XR Input Modality Manager

Component: XR Interaction Manager

Purpose: Coordinates all interaction events between Interactors (controllers, gaze) and Interactables (grabbable objects, UI). Processes Input Reader data in XRIT 3.0.

System 3: Event System with XR UI Input Module

GameObject: Event System

Components:

  • Event System - Unity's standard UI event system
  • XR UI Input Module - Extends Event System for XR ray interactions with UI

Purpose: Enables VR controllers to interact with Unity UI (buttons, sliders, etc.) using ray interactors.

System 4: XR Origin (XR Rig) - The Complete Player

GameObject: XR Origin (XR Rig)

Components on Root:

  • XR Origin - Defines VR world origin point
  • Character Controller - Allows physics-based collision and movement
  • Character Controller Driver - Syncs VR tracking with Character Controller
  • XR Input Modality Manager - Switches between controller and hand tracking
  • XR Gaze Assistance - Improves gaze interactor precision

Camera Offset and Main Camera (as covered in Part 2)

Child of XR Origin: Camera Offset

Children of Camera Offset:

  • Main Camera with Tracked Pose Driver (headset tracking)
  • Gaze Interactor (new addition in CW1 package) - see below
  • Left/Right Controller (enhanced versions) - see below

New Addition: Gaze Interactor System

üëÅÔ∏è What is Gaze Interaction?

Allows interaction by looking at objects - useful for accessibility, UI selection without controllers, and fallback interaction when hands are busy.

GameObject: Gaze Interactor (child of Camera Offset)

Components:

  • XR Gaze Interactor - Detects objects in center of view
  • XR Controller - Pre-populated with XRI Input Actions
  • Gaze Input Manager - Manages gaze-specific input (dwell time, etc.)

Sibling GameObject: Gaze Stabilized (with XR Transform Stabilizer) - Smooths gaze jitter

Enhanced Controller Setup: Full Interactor Suite

CW1 Controllers vs. Basic Near-Far Setup:

Instead of a single Near-Far Interactor, CW1 uses a complete suite of specialized interactors managed by an XR Interaction Group:

GameObject: Left/Right Controller (children of Camera Offset)

Components on Controller Root:

  • Action Based Controller Manager - Switches between controller schemes
  • XR Controller (Action Based) - Input actions reference (XRI Input Actions asset)
  • XR Interaction Group - Manages interaction priority between child interactors

Child Interactors (priority order):

  1. Poke Interactor
    • XR Poke Interactor component
    • Purpose: Touch-based UI interaction (poking buttons with fingertip)
  2. Direct Interactor
    • XR Direct Interactor component
    • Sphere Collider (Is Trigger enabled)
    • Purpose: Grab objects within reach using sphere collision detection
  3. Ray Interactor
    • XR Ray Interactor component
    • Line Renderer (visual ray line)
    • XR Interactor Line Visual (customizes ray appearance)
    • Sorting Group (UI rendering order)
    • Purpose: Point at and select distant objects and UI
  4. Teleport Interactor
    • XR Ray Interactor (configured for teleportation)
    • Purpose: Target teleportation anchors and areas

Sibling GameObject: Left/Right Controller Stabilized (with XR Transform Stabilizer) - Smooths hand tracking jitter
