The BB VR Package or your own project from week 01 is a good option for working with pre made or transferable elements. You should make your own, and ensure you can start or transfer VR projects quickly via Unity Packages.

The main workflow for 2026 Tutorial 02 is:

Part 1:

  1. Export your XR Rig Scene from Tutorial 01 as a Unity Package if it is basic – include dependencies.
  2. Create an XR Rig Prefab from the Tutorial 01 Scene if the Scene or Project has other work in it. You can export this Prefab with dependencies for a clean rig.
  3. In new or target project for Tutorial 02:
    1. Install OpenXR Package – fix any issues in Edit/Project Settings/XR Project Management/Project Validation.
    2. Import your XR Rig package – and depending on the package:
      1. Scene Package: A scene will import with the XR Rig in it.
      2. Prefab Package: Create a scene importing the prefab XR Rig.

Part 2:

  1. Import XR Interaction Toolkit.
  2. Import XRIT Samples:
    1. Starter Assets
    2. XR Device Simulator
    3. Legacy XR Input Readers

Part 2a:

  1. Import a Basic XRIT XR Rig from 2025 Package here – this will be basic and serve as a bridge from week 1 to this week.
    1. Add and Implement the Tracked Pose Driver (New Input System).
    2. Add Bindings to run controllers.
    3. This uses OpenXR and the XR Device Simulator to run, and gives a full set of controls.

    (Tutorial 01 was the Legacy Tracked Pose Driver with Generic Controllers run by the MockControllers.cs Script)

    (Tutorial 02 is new Tracked Pose Drivers using XR Input Actions)

Part 2b:

  1. Create a Standard XRIT XR Rig 2026 –
    1. This will have more fully featured inputs and interactions.
    2. Just use the basics for now – the rest will bring you to weeks 4 and 5 and CW1.
    3. Implement the Tracked Pose Driver (New Input System).
    4. Add Bindings to run controllers.
    5. This uses OpenXR and the XR Device Simulator to run, and gives a full set of controls.

    (Tutorial 01 was the Legacy Tracked Pose Driver with Generic Controllers run by the MockControllers.cs Script)

    (Tutorial 02 is new Tracked Pose Drivers using XR Input Actions)
